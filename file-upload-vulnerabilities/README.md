# file-upload-vulnerabilities

📌 مقدمة

هذا المستودع يشرح ثغرات رفع الملفات (File Upload Vulnerabilities) في تطبيقات الويب، وهي ثغرات تسمح للمهاجم برفع ملفات ضارة إلى السيرفر.

يتضمن المحتوى:

طرق الاستغلال الشائعة
أساليب تجاوز الحماية (Bypass Techniques)
التأثيرات الأمنية
أفضل طرق الحماية
🎯 الهدف من المشروع
فهم آلية رفع الملفات في تطبيقات الويب
اكتشاف الأخطاء الأمنية في هذه الخاصية
تطبيق الاستغلال عملياً
تعلم طرق الحماية بشكل صحيح
⚠️ خطورة ثغرات رفع الملفات

قد تؤدي هذه الثغرات إلى:

تنفيذ أوامر عن بعد (Remote Code Execution - RCE)
👉 يعني تشغيل أوامر على السيرفر من خارج النظام
رفع Web Shell
👉 ملف خبيث يعطي المهاجم تحكم كامل بالسيرفر عبر المتصفح
تخزين ملفات ضارة داخل النظام
تجاوز الصلاحيات (Privilege Escalation)
👉 الحصول على صلاحيات أعلى من المسموح بها
استخدامها كجزء من سلسلة هجمات (Attack Chain)
🧩 أنواع ثغرات رفع الملفات
1. ❌ رفع بدون تحقق (No Validation)
الوصف

التطبيق يسمح برفع أي ملف بدون أي فحص.

التأثير

يمكن رفع ملفات تنفيذية مثل:

.php
.jsp
.asp
النتيجة

تشغيل كود ضار مباشرة على السيرفر (RCE)

2. ⚠️ تحقق ضعيف (Client-Side Validation)
الوصف

التحقق يتم فقط في المتصفح (Frontend)، وليس في السيرفر.

مثال:

فحص الامتداد فقط
الاعتماد على Content-Type (نوع الملف في الطلب)
طرق التجاوز

باستخدام أدوات مثل Burp Suite:

تعديل Content-Type
تغيير اسم الملف:
shell.php.png
shell.jpg.php
المشكلة

أي تحقق يتم في المتصفح يمكن تجاوزه بسهولة.

3. 📂 رفع إلى مجلد غير قابل للتنفيذ
الوصف

يتم رفع الملف داخل مجلد لا يسمح بتنفيذ الكود بداخله.

مثال:

/uploads/
متى يصبح خطر؟

إذا وُجدت ثغرات أخرى مثل:

LFI (Local File Inclusion)
👉 ثغرة تسمح بتحميل ملفات من السيرفر وتشغيلها
سوء إعدادات السيرفر
4. 🚫 التصفية السوداء (Blacklist Filtering)
الوصف

منع امتدادات معينة فقط مثل:

.php, .jsp, .asp
طرق التجاوز
امتدادات بديلة:
.php5
.phtml
.phar
تغيير حالة الأحرف:
.PhP
امتدادات مزدوجة:
shell.php.jpg
المشكلة

القائمة السوداء (Blacklist) لا يمكن أن تكون كاملة.

5. ⚙️ استغلال إعدادات السيرفر
الوصف

رفع ملفات تتحكم في طريقة تعامل السيرفر مع الملفات.

أمثلة
Apache (.htaccess)
AddType application/x-httpd-php .jpg

👉 يجعل ملفات .jpg تُنفذ كـ PHP

IIS (web.config)

👉 تغيير إعدادات التنفيذ داخل السيرفر

التأثير

تحويل ملفات عادية إلى ملفات قابلة للتنفيذ

6. 🧬 إخفاء الامتداد (Obfuscation)
الوصف

إخفاء طبيعة الملف الحقيقي لتجاوز الفلاتر

الطرق
ترميز الأحرف (Encoding)
امتدادات مزدوجة
Null Byte (في أنظمة قديمة)
امتدادات بديلة:
.php5
.phtml
الهدف

خداع أنظمة الفحص البسيطة

7. ⏱️ Race Condition
الوصف

وجود فرق زمني بين:

رفع الملف
فحصه
تخزينه أو حذفه
الاستغلال

رفع ملف خبيث ومحاولة الوصول إليه قبل أن يتم حذفه

🛡️ طرق الحماية (Mitigation)
التحقق من الملفات في السيرفر فقط (Backend)
استخدام Allowlist
👉 السماح فقط بامتدادات محددة (مثل .jpg, .png)
إعادة تسمية الملفات عشوائياً
تخزين الملفات خارج Web Root
👉 مجلد لا يمكن الوصول له مباشرة من المتصفح
تعطيل تنفيذ الملفات داخل مجلد الرفع
فحص محتوى الملف (وليس الاسم فقط)
استخدام Web Application Firewall (WAF) كطبقة حماية إضافية
📌 الخلاصة

ثغرات رفع الملفات تحدث بسبب:

ضعف التحقق من الملفات
سوء تخزينها
أو السماح بتنفيذها داخل السيرفر

وأي خطأ بسيط في هذه المراحل قد يؤدي إلى اختراق كامل للنظام.
